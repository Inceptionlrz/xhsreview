# XHS Review · 小红书智能回复助手

> 仿 Linux.do 刷帖助手 v8.5.0 风格的桌面工具：自动浏览小红书首页推荐流，**基于 Anthropic Messages API 动态生成回复**，可自定义 URL 与 Key。

**⚠️ 重要风险提醒（必读）**

本工具本质是浏览器自动化脚本，**小红书官方明确禁止**使用任何第三方工具、脚本或 AI 自动浏览、查看、点赞、评论。已有用户收到「账号违规预警」：

- 提示「请求次数过多」后收到系统消息；
- 被平台判定为「疑似使用三方工具或脚本（如 AI）自动运营账号」；
- 首次多为警告，多次可能限制账号功能。

**请把本工具仅用于学习研究，并自行承担全部风险。** 真实账号运行前，建议先用小号/弃号验证，且必须开启「🕐 运营画像」中的每日上限与活跃时段。

## ✨ 功能

- 🧠 **AI 智能回复**：调用 Anthropic Messages API，针对每个帖子的标题/正文生成自然口语化的回复
- ⚙️ **API 完全可配**：URL、Key、Model、人设、温度、max_tokens 全部可自定义，支持中转站
- 🌐 **真实抓取**：基于 Playwright 浏览器自动化，模拟真人滚动/键入，反检测
- 🔁 **虚拟数据模式**：未安装 Playwright 或无登录态时使用，零依赖可演示
- 📊 **三大运行模式**：无限制 / 帖子数 / 时间限制
- 🐢 **双爬楼模式**：深度爬楼（完整阅读）/ 快速浏览（3~5 层换帖）
- 👍 **行为概率控制**：自动点赞率、自动回复率、随机等待区间
- 🛡️ **拟人化 / 防封（核心）**：
  - 逐字随机打字 + 偶发卡顿 + 偶发「打错重打」
  - 鼠标曲线靠近 + 过冲修正 + 犹豫不点
  - 评论前阅读停留 + 先下滑看已有评论
  - 纯浏览跳过、随机长休息、会话操作上限、顺序随机
  - 环境指纹伪装（WebGL 渲染器、navigator.plugins、languages 等）
  - AI 回复内容后处理（偶发错别字 / emoji / 截断 / 标点波动 / 开头语气词）
  - **通用反应分支**：一定比例直接发真人常用口语反应（前排 / 蹲一个 / 收藏了 / 单个 emoji…），**跳过 AI 生成**——这是抗「AI 运营」检测最有效的手段，此类低信息评论几乎不可能被判为 AI
  - **人设轮换**：从「人设池」随机选人设，打破「单一完美声音」指纹
  - **语气随机化**：每次生成随机切换（纯情绪 / 反问 / 含糊 / 共情 / 切题），不再总是写切题好评
  - **相关性闸门**：帖子正文过短强制走通用反应（没料可评，硬评反而露馅）
- 🕐 **运营画像 / 风控**：
  - 每日点赞 / 评论上限（跨重启累计）
  - 只在指定活跃时段运行（默认 09:00-23:00）
  - 两次运行最小间隔
- 🌱 **养号模式（冷启动）**：账号被平台判定为 AI 运营后的「养号」开关（主界面「🌱 养号模式」勾选框）。开启后自动叠加：每日≤3 次极低频点赞、85% 帖子纯浏览跳过、**绝不评论**、帖子间等待 8~20 秒、长休息概率 35% 且单次 2~7 分钟、会话操作上限 4。仅影响本次运行，不回写配置文件，关掉即刻恢复原行为。
- 🧩 **板块筛选**：内置多个板块（推荐、穿搭、美食、彩妆等）
- 📈 **实时统计**：帖子 / 爬楼 / 点赞 / 回复 / 错误 / 升级进度
- 🔑 **激活授权**：三档激活码（试用 1 天 / 月卡 30 天 / 永久），与安装机器码绑定，未授权、过期或换机器均无法运行

## 🔑 激活授权

软件使用 **机器码绑定的激活码** 做离线授权，无需联网验证，杜绝同一激活码在多台机器复用。

### 授权档位

| 档位 | 参数值 | 有效期 | 说明 |
|---|---|---|---|
| 试用 | `day` | 1 天 | 体验用，过期后需重新激活 |
| 月卡 | `month` | 30 天 | 到期前需续费 |
| 永久 | `permanent` | 永久（无过期时间） | 一次激活，永久可用 |

### 用户激活流程

1. 打开软件 → 点击用户信息栏的「**激活**」按钮
2. 弹窗中显示本机 **机器码**（如 `3312-2D0C-5406-9A1D`），点击复制后发给开发者
3. 开发者用生成器生成对应档位的激活码（见下文）
4. 将激活码粘贴回弹窗 → 校验通过后永久写入 `config/license.json`，下次启动免输入

### 技术原理（防伪造）

- **机器码**：由本机硬件指纹（CPU / 主板 / BIOS / 磁盘序列号 + 首网卡 MAC）哈希生成，格式 `XXXX-XXXX-XXXX-XXXX`，跨重启稳定
- **激活码结构**：`XHS1.<payload>.<signature>`，payload 内含 `{版本, 档位, 机器码, 过期时间, 签发时间}`
- **签名**：RSA-2048 非对称签名（SHA-256）。**私钥只在开发者侧**（`scripts/license_private_key.pem`），**公钥内置** `src/license.py` 仅用于验签 → 用户无法自行伪造
- **绑定校验**：激活码内机器码与本机不符 → 拒绝；过期 → 拒绝；签名被篡改 → 拒绝
- 激活码支持 `-` / 空格 分组输入（便于复制粘贴），程序会自动归一化

### 开发者：生成激活码

```bash
# 语法
python scripts/gen_license.py --machine <机器码> --tier day|month|permanent

# 示例：给机器码 3312-2D0C-5406-9A1D 生成永久码
python scripts/gen_license.py --machine 3312-2D0C-5406-9A1D --tier permanent
```

输出为分组后的激活码字符串，直接发给用户即可。

> 🔒 **私钥安全**：`scripts/license_private_key.pem` 已被 `.gitignore` 排除，**绝不可提交或随软件分发**。一旦泄露，任何人都能自行签发激活码。若怀疑泄露，需更换密钥对并重新发放所有激活码。

## ⚙️ 首次使用

1. 点击「⚙ Anthropic 配置」填入：
   - **API URL**：默认 `https://api.anthropic.com`（支持中转站）
   - **API Key**：你的 Key
   - **Model**：`claude-3-5-sonnet-20241022` 或其他支持 Messages API 的模型
   - **人设**：AI 回复风格描述
2. 切换「虚拟数据 / 真实抓取」：
   - 默认 **🔁 虚拟数据：开**（无需登录，开箱即用）
   - 真实抓取需先扫码登录一次（profile 目录保留登录态）
3. 打开「🛡 拟人化设置」和「🕐 运营画像」，按下列推荐保守值调整（尤其是收到警告后）。
4. 点击「▶ 开始」

## 🛡 收到警告后的推荐保守配置

如果你已经收到「请求次数过多」或「账号违规预警」，请先把参数压到保守档，观察 1-2 天再决定是否缓慢上调：

| 设置项 | 推荐值 | 说明 |
|---|---|---|
| 自动点赞 | 关闭 / 开启 | 警告后建议关闭，或点赞率 ≤ 10% |
| 点赞率 | 10% | 多数帖子只浏览 |
| 自动回复 | 开启 | 但回复率 ≤ 25% |
| 回复率 | 25% | 大量帖子只看不评 |
| 帖子间等待 | 3~8 秒 | 不要低于 3 秒 |
| 纯浏览跳过率 | 45% | 近一半帖子完全不操作 |
| 看了但不评论率 | 25% | 连 AI 调用都省，更自然 |
| **通用反应率** | **45%** | **关键：一定比例只发口语反应（前排/蹲一个/收藏了…）跳过 AI，最抗「AI 运营」检测** |
| **人设池** | 填 3~5 条不同人设 | **开启人设轮换，打破单一声音指纹** |
| **正文过短强制通用反应** | 25 字 | 内容太短没料可评，硬评反而露馅 |
| 会话操作上限 | 20 次 | 达到后强制长休 3~10 分钟 |
| 每日点赞上限 | 15 次 | 跨重启累计 |
| 每日评论上限 | 12 次 | 跨重启累计 |
| 活跃时段 | 09:00-23:00 | 避免深夜/凌晨运行 |
| 两次运行最小间隔 | 30 分钟 | 防止反复停止-启动刷量 |
| 长休息概率 | 15% | 每帖处理后有机会长休 45~180 秒 |

> 这些默认值已在 `config.example.json` 和 GUI「恢复默认」中生效。如果你之前保存过旧配置，建议手动把 `config/config.json` 里的 `like_rate`、`reply_rate`、`skip_rate` 等改到上表范围，或在对应弹窗点「恢复默认」。

## 📦 打包成 EXE

本项目已配置 `xhsreview.spec`，使用 PyInstaller 的 `onedir` 模式，打包时会把 Playwright 浏览器随包携带，目标机无需再安装 Python 或 Playwright。

推荐用一键脚本：

```bash
python scripts/build_exe.py
```

或直接：

```bash
python -m PyInstaller xhsreview.spec --noconfirm
```

打包产物在 `dist/xhsreview/xhsreview.exe`。**打包时务必确保 `config/config.json` 不被包含**，因为它包含真实 API Key。`xhsreview.spec` 只包含 `config.example.json`。

> 🔐 **授权打包注意**：
> - `xhsreview.spec` 已加入 `hiddenimports=['cryptography', 'src.license']`，否则打包后 exe 在激活码校验时会因缺少 `cryptography` 库崩溃。
> - **私钥 `scripts/license_private_key.pem` 绝不可打包进 exe**（它已被 `.gitignore` 排除，但打包脚本也需确认不将其作为 `datas` 带入）。公钥已硬编码在 `src/license.py` 中随程序分发，私钥只留在开发者本机。
> - `config/license.json`（用户激活记录）同样不可打包，否则所有分发副本会共享同一台机器的授权状态。
